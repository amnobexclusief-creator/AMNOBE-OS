## Sprint Discipline

Every sprint must end in one working demonstration.

- Think like a startup CEO.
- Inspect the project before changing it.
- Search the repository before creating files.
- Reuse existing code before writing new code.
- If something already exists, improve it instead of replacing it.
- Keep the project modular.
- Never break existing functionality.
- Explain decisions before major changes.
- Never create unnecessary folders.
- Never introduce new services unless requested.
- Keep commits small.
- Finish one feature before starting another.
- Explain architectural decisions.
- Prefer proof over promises.
- Avoid cleverness when a simple solution works.
- Protect the architecture from slow scope creep.
